Enhanced `Texture2D` grayscale conversion to gracefully handle non-RGBA modes by converting to RGBA first. Broadened `Material` color matching to target any property containing "color", "tint", or "glow" to catch all custom shader effects that were missing colors.
*   **Color Changer String Replacement Bug & Recursive Coverage**: Fixed a critical bug where `colorChanger` mode was replacing internal strings (like `first` paths) with the hex color string (e.g., `#000000`). By forcing `pattern = None` during color change mode, it now strictly modifies colors and the `AssetBundle` watermark without touching internal structural strings. Also implemented a `replace_all_colors_recursive` function to aggressively target *all* `ColorRGBA` structures inside `ParticleSystem` and `MonoBehaviour` trees, and removed name filters for `Material` colors, ensuring 100% color coverage.
*   **Large File Processing Optimization (Fast Typetree Reading)**: Fixed a major performance bottleneck where processing large 7MB+ files in `colorChanger` mode was extremely slow. The script previously read and parsed the typetree for every single object (GameObjects, Animators, Transforms, etc.) even when no string replacement was needed. We optimized `process_bundle.py` to strictly skip `obj.read_typetree()` for non-essential objects when `pattern` is `None`. Now it only reads typetrees for targeted objects (`Material`, `ParticleSystem`, `Image`, etc.), reducing processing time for large hero skins from minutes to seconds.
*   **Color Changer Checkpoint Restore (Sledgehammer Approach)**: Reverted recent attempts to fix the "abu-abu" (gray mesh) and transparent/square box particle issues (`ImageOps.colorize`, `_SrcBlend`/`_DstBlend` manipulation, and `ImageChops` multiply logic). Those techniques, while attempting to fix flat gray meshes and invisible black additive effects, ended up breaking other intricate particle thickness layers and causing transparent effects in-game. We have strictly restored the working **Sledgehammer** state: it simply converts `Texture2D` to pure Grayscale (retaining original Alpha) and relies on the recursive tree search to inject the target Hex color directly into all `ColorRGBA` structures (Material properties, ParticleSystem modules, MonoBehaviour colors). This is the safest, most stable baseline that guarantees colors are changed globally without breaking effect thickness.
*   **LZ4 Compression Toggle**: Added a user-facing toggle in the UI (below the watermark string input) allowing the user to select whether to compress the output AssetBundle using LZ4 or skip compression. **CRITICAL FIX**: When skipping compression, we must explicitly use `env.file.save(packer="none")`. If left empty (`env.file.save()`), UnityPy falls back to the original bundle's compression (often LZMA), which causes massive memory spikes (OOM corruptions in AI Studio) and extremely slow processing times (15+ minutes in Hugging Face). Connected through `server.ts` and `process_bundle.py` arguments.
*   **Background Processing Queue (Timeout & Corruption Fix)**: We fully migrated the `/api/process-batch` Express endpoint to a background job polling model. Instead of a single blocking 15-minute HTTP request that would cause 504 Gateway Timeouts on HuggingFace and AI Studio (resulting in corrupt/incomplete ZIPs), the Express server now instantly returns a `jobId` and processes files asynchronously in memory. The React client polls `/api/job-status/:jobId` every 2 seconds, and hits `/api/job-download/:jobId` only once the server confirms `.status === 'completed'`. This guarantees 100% stability for massive jobs without Nginx/Proxy disconnects.
*   **Download Progress Tracker**: Enhanced the file download mechanism on the client-side. Instead of a static "Downloading processed files..." message while the browser downloads the heavy ZIP file silently, we implemented a `ReadableStream` reader using `fetch` and `blob()`. The UI now dynamically displays the download progress percentage (if `content-length` is available) or the loaded MBs in real-time. This provides clear feedback to the user that the server is actively sending data and hasn't frozen.
*   **Smart Processing Optimization (Ganti Skin)**: Optimized "Ganti Akhiran / Skin Spesifik" to only parse and process 2 targeted object types (`AssetBundle` and `GameObject`), significantly boosting processing speed for large hero bundles.
*   **"first" Key Lowercase Fix**: Restored the critical logic ensuring that the `first` key's value inside the typetree dictionary is automatically lowercased (`target_str.lower()`) to maintain structure compatibility.

We added `MonoBehaviour` to the allowed list for `mode == "all"`, restoring effect replacement functionality while maintaining the speed optimization.
